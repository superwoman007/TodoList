# TodoList 产品需求文档（PRD）

## 1. 背景与目标
- 为个人与小团队提供美观、轻量、高可用的任务管理工具，支持 Web 与移动端（PWA 可安装），后端采用 Python。
- 关键价值：快速记录与组织任务、随时随地访问、离线工作、良好交互与可访问性。

## 2. 用户画像
- 个人效率用户：需要快速记录、分类与提醒。
- 轻量协作用户：需要基本共享与活动记录（后续扩展）。

## 3. 范围（MVP）
- 任务：创建/编辑/删除、状态（待办/进行中/完成）、优先级、截止日期。
- 标签：创建/绑定/过滤。
- 子任务：层级仅一层，支持勾选完成。
- 列表与看板：列表视图与按状态列的看板视图（拖拽）。
- 搜索与过滤：关键字、标签、状态、截止日期、优先级。
- 提醒：到期提醒（后台调度）。
- 账户与鉴权：注册、登录、退出；个人设置。
- 离线：缓存任务列表与基本操作；在线时同步。

## 4. 非功能性需求
- 性能：首屏 ≤ 2s（本地预览），交互响应 150–300ms 动画。
- 可访问性：对比度≥4.5:1，键盘导航与可见焦点。
- 可靠性：乐观更新与冲突解决（以服务器 updatedAt 为准）。
- 安全：JWT 鉴权、输入校验、速率限制、密钥安全。

## 5. 平台与技术
- 前端：Next.js 15 + TypeScript + Tailwind + shadcn/ui + TanStack Query + Zustand。
- 后端：FastAPI + SQLAlchemy + Alembic；DB：Postgres/SQLite；调度：APScheduler。
- 移动端：PWA（iOS/Android 安装）；可选 React Native + Expo（后续阶段）。

## 6. 用户故事与验收标准（选摘）
1) 作为用户，我可以创建任务并设置优先级与截止日期。\n
   - 验收：点击“添加任务”→ 输入标题（必填）→ 可选描述/优先级/截止日期 → 保存成功且任务出现在列表顶端。\n
2) 我可以为任务添加标签，并在列表中按标签过滤。\n
   - 验收：标签选择器支持创建与绑定；过滤后仅显示含选中标签的任务。\n
3) 我可以在看板视图通过拖拽改变任务状态。\n
   - 验收：拖拽到目标列后状态更新，并在列表与详情中保持一致；移动端长按拖拽可用。\n
4) 我可以在离线时查看与编辑任务，恢复在线后自动同步。\n
   - 验收：离线模式可访问缓存数据；在线后增量同步，无数据丢失。\n
5) 我可以收到到期提醒。\n
   - 验收：后台调度到期触发（本地通知或站内提示），提醒状态可查看与关闭。\n

## 7. 指标与门禁
- 覆盖率≥70%；A11y 检查通过；无高危 Bug；移动端断点适配良好（375/768/1024/1440）。

## 8. 交付物
- UI 规范与页面原型；前后端代码；测试与报告；预览与部署链接。
